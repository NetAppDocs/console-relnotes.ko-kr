=== 콘솔 설정 및 관리



==== Console 에이전트 4.6.0

4.6.0 릴리스는 표준 모드와 제한된 모드를 모두 지원합니다.

이번 Console 에이전트 릴리스에는 보안 개선 사항 및 버그 수정 사항과 더불어 다음과 같은 새로운 기능이 포함되어 있습니다.

.필요한 디스크 공간의 구성 확인 지원
구성 검사를 사용하여 Console 에이전트에 필요한 디스크 공간을 사용할 수 있는지 확인할 수 있습니다. Console 또는 Agent 유지 관리 콘솔에서 구성 검사를 수행할 수 있습니다.link:https://docs.netapp.com/us-en/console-setup-admin/task-troubleshoot-agent.html["Console 에이전트에서 구성 검사를 실행하는 방법에 대해 자세히 알아보세요"^]

.ISO를 사용하여 vCenter에 배포된 Console 에이전트 업그레이드
OVA를 사용하여 vCenter에 Console 에이전트를 배포한 경우, 이제 ISO를 VM에 마운트하고 VM 유지 관리 콘솔에서 업그레이드를 실행하여 새 버전으로 업그레이드할 수 있습니다. link:https://docs.netapp.com/us-en/console-setup-admin/task-upgrade-agent.html["vCenter에 배포된 콘솔 에이전트를 업그레이드하는 방법을 알아보세요."^]

보호 정책에 대한 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-policies-create.html["NetApp Backup and Recovery에서 보호 정책을 생성하고 관리하기"]을 참조하십시오.

Kubernetes 워크로드 보호에 대한 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-kubernetes-protect-overview.html["Kubernetes 워크로드 보호 개요"]을 참조하세요.

==== KVM 워크로드 개선 사항

* *작업 처리 기능 개선*: NetApp Backup and Recovery가 이제 여러 개의 동시 작업을 더욱 효율적으로 처리합니다.

==== VMware 워크로드 개선 사항

이번 릴리스에서 VMware 워크로드는 다음과 같은 개선 사항을 도입했습니다.

* *태그 기반 보호*: VMware 워크로드용 백업 및 복구는 이제 NFS 및 VMFS 볼륨에 저장된 VM에 대한 태그 기반 보호를 지원하므로 개별 VM 또는 데이터스토어 대신 vCenter 태그를 기반으로 보호 그룹을 구성할 수 있습니다. vCenter에서 VM 및 데이터스토어에 태그를 지정하면 백업 및 복구가 자동으로 해당 항목을 검색하고 수동 그룹 유지 관리 없이 향후 백업에 포함합니다. 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-protection-groups.html["보호 그룹 생성 및 관리"]을 참조하십시오.
* *대체 스토리지 시스템으로 복원*: 이제 원래(운영) 스토리지 시스템을 완전히 사용할 수 없는 경우에도 가상 머신을 대체 스토리지 시스템으로 복원할 수 있습니다. 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore.html["VMware 워크로드 복원"]을 참조하십시오.
* *Linux VM에 파일 복원*: 이제 VM 파일과 폴더를 Linux VM에 복원할 수 있습니다. 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore-guest-files-and-folders-from-vmdks.html["파일 및 폴더 복원"]을 참조하십시오.
* *파일 및 폴더 복원 시 프록시 VM 지원* - 이제 소스 vCenter에서 호스팅되는 다른 VM으로 파일 및 폴더를 복원할 수 있습니다. 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore.html["VMware 워크로드 복원"]을 참조하십시오.
* *재귀적 파일 및 폴더 검색* - 이제 VMDK 파일 전체에서 특정 파일이나 폴더를 검색할 수 있습니다. 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-restore.html["VMware 워크로드 복원"]을 참조하십시오.
* *작업 처리 기능 개선*: NetApp Backup and Recovery가 이제 여러 개의 동시 작업을 더욱 효율적으로 처리합니다.


VMware 워크로드 보호에 대한 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-protect-overview.html["VMware 워크로드 보호 개요"]을 참조하십시오.

=== 재해 복구



==== Google Cloud VMware Engine(GCVE) 지원이 이제 정식 출시(GA)되었습니다.

NetApp Disaster Recovery의 GCVE 지원이 이제 일반 공급(GA)으로 제공됩니다. GA 지원을 통해 GCVE용 Disaster Recovery는 이제 다음을 지원합니다:

* *플랫폼 관리형 스냅샷:* ONTAP 또는 Google Cloud Platform(GCP)을 통해 백업 및 보존 정책을 원활하게 관리합니다.
* *복제 계획당 여러 데이터 저장소:* 여러 데이터 저장소를 단일 복제 전략으로 통합하여 재해 복구 계획을 간소화하십시오.
* *사용할 수 없는 소스 사이트에 대한 강제 페일오버:* 소스를 사용할 수 없는 경우 재해 복구 사이트로 전환합니다.
* *SnapMirror 재동기화 연기 및 필요에 따라 복제 복원:* "보호 건너뛰기" 옵션을 사용하여 운영량이 많은 시간대에 프로덕션 워크로드에 미치는 영향을 방지하기 위해 SnapMirror 재동기화를 연기하고, 나중에 재보호 옵션을 사용하여 필요에 따라 복제를 복원할 수 있습니다


GCVE 지원에 대한 자세한 내용은 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/release-notes/dr-limitations.html["Disaster Recovery를 위한 지원 구성"^]을(를) 참조하십시오.

=== 랜섬웨어 복원력



==== Google SecOps용 SOAR 플레이북

Ransomware Resilience는 이제 link:https://github.com/chronicle/content-hub/tree/main/content/response_integrations/third_party/partner/netappransomwareresilience["Google SecOps"^]에 대한 보안 오케스트레이션, 자동화 및 대응(SOAR) 플레이북을 제공합니다. 이 플레이북을 통해 위협 대응 관리가 더욱 간편해지고, 작업 및 테스트가 자동화되며, 스냅샷 생성 및 볼륨 오프라인 전환과 같은 필수적인 랜섬웨어 대응 활동을 지원할 수 있습니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/reference-soar.html["SOAR 플레이북 통합"^]를 참조하십시오.

==== Cloud Volumes ONTAP에 대한 Console 에이전트 프록시 설정을 관리하는 기능

이제 연결이 끊어지거나 프록시 설정이 잘못된 경우에도 NetApp Console 에이전트를 사용하여 Cloud Volumes ONTAP의 프록시 설정을 관리할 수 있습니다. 이전에는 Console 에이전트가 20분 동안 Cloud Volumes ONTAP에 연결할 수 없는 경우 수동 프록시 설정을 기본 설정으로 덮어썼습니다. 이로 인해 AutoSupport 메시지 관련 문제를 포함한 통신 오류가 발생했습니다. 기존 시스템의 프록시 설정을 유지하려면 다음 API 호출을 실행하십시오.

`PUT /occm/config`

요청 본문에 다음 매개변수를 포함하십시오.

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
기본 모드는 표준 모드입니다. 즉, Console 에이전트가 20분 동안 Cloud Volumes ONTAP에 연결할 수 없는 경우 프록시 설정을 기본값으로 덮어씁니다.

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["수정 가능한 NetApp Console 매개변수를 구성합니다"^]

=== 데이터 분류



==== 버전 1.52

.리소스 유효성 검사
안정성을 보장하기 위해 Data Classification은 스캔, 보고서 생성 및 작업 전반에 걸쳐 사용 가능한 메모리 및 디스크 리소스를 검증합니다. 메모리 사용량이 95%를 초과하거나 디스크 공간이 6GB 미만으로 떨어지면 Data Classification은 자동으로 작업을 일시 중지합니다. 리소스가 충분해지면 작업이 재개됩니다.

시스템 리소스를 모니터링하려면 link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["Data Classification의 상태를 모니터링하세요"^]을 참조하십시오.

.복사 및 이동 제한 증가
이제 CIFS 대상에 대한 복사 및 이동 기능이 지원됩니다. 두 작업 모두 파일당 크기 제한이 기존 50MB에서 1GB로 증가했습니다. 두 작업 모두 작업량 제한이 1TB 또는 500만 파일로 늘어났습니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["데이터 복사, 이동 및 삭제"^]를 참조하십시오.

.Active Directory 통합 구성 권한
Data Classification에서 Active Directory(AD) 통합 설정 변경은 이제 관리자에게만 허용됩니다. 뷰어는 더 이상 AD 설정을 수정할 수 없습니다.

분류 뷰어 역할에 대한 자세한 내용은 link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["데이터 서비스 역할"^]을 참조하십시오.

.RHEL 9.7 지원
Data Classification에 Red Hat Enterprise Linux v9.7이 추가되어 다크 사이트 배포를 포함한 수동 온프레미스 설치가 가능합니다.

다음 운영 체제는 Podman 컨테이너 엔진을 사용해야 하며 Data Classification 버전 1.30 이상이 필요합니다: Red Hat Enterprise Linux 버전 8.8, 8.10, 9.0, 9.1, 9.2, 9.3, 9.4, 9.5, 9.6 및 9.7.

.투명 프록시 지원
Data Classification은 이제 투명 프록시 구성을 지원하여 프록시 또는 제한된 네트워크 환경에서 더욱 원활한 배포를 가능하게 합니다.

제한된 환경에서의 배포에 대한 자세한 내용은 link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-compliance-dark-site.html["인터넷 연결이 없는 Linux 호스트에 NetApp Data Classification 설치"^]을 참조하십시오.

=== Cloud Volumes ONTAP



==== 차세대 Google Cloud VM을 위한 Cloud Volumes ONTAP 지원

9.18.1에서 NetApp은 새로운 Cloud Volumes ONTAP 배포를 N2 VM에서 차세대 Google Cloud C3 시리즈 VM으로 전환하여 더욱 빠르고 확장 가능한 환경을 제공합니다. 이제 Google Cloud에서 Cloud Volumes ONTAP 9.18.1 이상 버전을 배포할 때 C3 시리즈 VM의 이점을 활용할 수 있습니다. C3 시리즈 머신은 Google Virtual NIC(gVNIC)와 Hyperdisk Balanced 디스크를 사용하여 향상된 성능과 더 높은 용량 제한을 제공하며, 집약적인 워크로드에 필요한 동적 성능을 보장합니다.


NOTE: 현재 Cloud Volumes ONTAP는 단일 노드 배포 환경에서만 C3 시리즈를 지원합니다.

Cloud Volumes ONTAP 시스템이 9.18.1 이상을 실행하는 경우 간편한 단일 노드 배포에 사용하는 사전 구성된 패키지는 자동으로 C3 VM을 사용하며 워크로드 요구 사항에 따라 IOPS 및 처리량 매개 변수를 사용자 지정할 수 있습니다. 마찬가지로 애그리게이트를 생성하는 동안 Hyperdisk Balanced 디스크를 추가하여 Google Cloud에서 더 나은 성능과 확장성을 달성할 수 있습니다. 또한 기본 Flash Cache 지원을 위해 C3 시리즈 머신의 LSSD 변형을 선택할 수 있습니다.

C3 VM은 Hyperdisk Balanced 디스크만 지원하기 때문에 애그리게이트에 볼륨을 추가할 때 디스크 유형을 변경할 수 없습니다. 마찬가지로 N2 VM 유형의 시스템을 C3 VM으로 복제할 때 디스크 유형은 기본적으로 Hyperdisk Balanced로 설정됩니다.

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud에서 Cloud Volumes ONTAP에 대해 지원되는 구성"^]

https://docs.cloud.google.com/compute/docs/general-purpose-machines#c3_series["Google 문서: C3 머신 시리즈"^]



==== Azure의 Cloud Volumes ONTAP용 VNet 보안

Azure 단일 및 다중 가용성 영역에 배포된 Cloud Volumes ONTAP 9.18.1 이상 버전은 전송 중인 데이터를 보호하기 위한 계층형 보안 전략의 일환으로 Azure 가상 네트워크(VNet) 암호화를 지원합니다. Cloud Volumes ONTAP는 Azure의 기본 데이터그램 전송 계층 보안(DTLS) 프로토콜을 활용하여 ONTAP 노드, 관리 인터페이스 및 기타 Azure 서비스 간의 통신을 보호하고, 데이터 가로채기 및 무단 액세스를 방지합니다. 이러한 네트워크 수준 암호화는 ONTAP에 내장된 스토리지 및 저장 데이터 보호 기능을 보완하여 데이터에 대한 엔드 투 엔드 보안을 제공합니다.

https://review.docs.netapp.com/us-en/storage-management-cloud-volumes-ontap_gcp-sn-c3-support/reference-networking-azure.html#networking-for-azure-vnet-encryption["Azure VNet 암호화를 위한 네트워킹"^]

* *클러스터 마이그레이션*: 이제 Trident Protect로 보호되는 기존 Kubernetes 클러스터를 Backup and Recovery에 추가할 수 있습니다.

* *복원 탭 통합*: 이제 복원 메뉴에서 Kubernetes 워크로드 복원 작업에 액세스할 수 있습니다.
* *3-2-1 팬아웃 백업 아키텍처 지원*: 이제 Kubernetes 워크로드를 보호할 때 보호 정책에서 3-2-1 팬아웃 아키텍처를 사용할 수 있습니다.

==== Oracle Database 워크로드 개선 사항

이번 Oracle Database 워크로드 릴리스에서는 다음과 같은 향상된 기능이 도입되었습니다.

* *루트 권한이 없는 사용자 지원*: 이제 루트 권한이 없는 사용자도 백업, 복원 및 복제 작업을 수행할 수 있어 보안 및 규정 준수가 향상됩니다.
* *클론 지원*: 이제 ASM 라이브러리 v2를 사용하여 운영 및 2차 NAS, SAN 및 ASM 환경 전반에 걸쳐 클론 기능을 지원하여 통합 보호 워크플로를 구현할 수 있습니다.
* *클론 분할 지원*: 이제 쓰기 가능한 스냅샷(클론)을 상위 볼륨에서 분할하여 스토리지를 확보하고 독립적인 작업을 수행할 수 있습니다.
* *객체 저장소에 대한 백업 및 복원*: 이제 객체 기반 S3 호환 스토리지 타겟에 대한 네이티브 백업 및 복원 기능이 지원됩니다.
* *Clone Lifecycle Management(CLM)*: 운영 스토리지에서 클론 새로 고침 작업이 지원됩니다.
* *다른 호스트로 복제*: 이제 기본 스토리지와 2차 스토리지 모두에서 데이터베이스를 다른 호스트로 복제할 수 있습니다(테스트 또는 분석용).
* *ONTAP 일관성 그룹 지원*: 이제 ONTAP 일관성 그룹이 지원되어 여러 볼륨에서 애플리케이션 정합성 보장 스냅샷을 보장합니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-data-classification/task-copy-move.html["데이터 복사, 이동 및 삭제"^]를 참조하십시오.

==== 포괄적인 보안 감독을 위한 새로운 보안 보고서

Digital Advisor에는 이제 스토리지 환경 전반의 ONTAP 보안 위험에 대한 통합 보기를 제공하는 보안 보고서가 포함되어 있습니다. 이 보고서를 통해 클러스터, SVM 및 볼륨 전반에 걸쳐 잘못된 구성, 규정 준수 격차 및 보안 문제를 더 빠르게 감지하여 안전한 스토리지 인프라를 유지할 수 있습니다.



==== 패치 버전이 혼합된 환경에 대한 Upgrade Advisor 지원 기능이 향상되었습니다.

Upgrade Advisor는 이제 동일한 ONTAP 주요 릴리스 내에서 노드가 서로 다른 패치 레벨을 실행하는 클러스터에 대한 업그레이드 보고서를 생성합니다. 이 개선 사항을 통해 고객은 클러스터가 주요 버전 간에 혼합되지 않는 한 원활하게 업그레이드를 진행할 수 있으며, 업그레이드 계획에 있어 더 큰 유연성을 제공합니다.



==== 지원 기간이 종료된 ONTAP 버전에 대한 지원이 확장되었습니다.

Upgrade Advisor는 이제 제한적 지원이 종료된 ONTAP 버전에 대한 업그레이드 계획 생성을 지원하며, 9개월의 유예 기간을 제공합니다. 이 기능은 전환 기간 동안 명확한 지침을 제공하여 고객이 적시에 조치를 취하고 업그레이드 중단을 방지할 수 있도록 지원합니다.

NOTE: 이번 릴리스에서는 작업 메뉴의 *업그레이드* 옵션을 사용하여 Hyper-V 또는 Windows용 NetApp 플러그인을 업그레이드할 수 없습니다. 그 대신 각 Hyper-V 호스트를 제거한 다음 다시 추가하여 플러그인을 업데이트하십시오.

https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-hyperv-restore.html["NetApp Backup and Recovery 사용하여 Hyper-V VM을 복원하는 방법에 대해 자세히 알아보세요."].

== 2026년 1월 14일



=== 데이터 분류



==== 버전 1.50

이번 Data Classification 릴리스에는 버그 수정과 다음 업데이트가 포함되어 있습니다.

.사용자 지정 분류 개선
이제 데이터 분류 기능에서 사용자 지정 범주 생성을 지원합니다. 파일을 업로드하여 데이터 분류에서 데이터에 범주 마커를 적용하는 데 사용하는 AI 모델을 세밀하게 조정할 수 있습니다. 모든 사용자 지정 분류에 대한 인터페이스가 개선되었습니다.

자세한 내용은 다음을 참조하세요. link:https://docs.netapp.com/us-en/data-services-data-classification/task-custom-classification.html["사용자 정의 분류 만들기"^].

.사용자 지정 오래된 데이터 정의
데이터 분류 기능을 통해 이제 조직의 요구 사항에 맞게 오래된 데이터의 정의를 사용자 지정할 수 있습니다. 이전에는 오래된 데이터를 마지막으로 수정된 지 3년이 지난 데이터로 정의했습니다. 이제 오래된 데이터는 마지막 접근 시점 또는 마지막 수정 시점을 기준으로 식별할 수 있으며, 그 기간은 6개월 전부터 10년 전까지 다양할 수 있습니다.

자세한 내용은 다음을 참조하세요. link:https://docs.netapp.com/us-en/data-services-data-classification/task-stale-data.html["오래된 데이터 정의를 사용자 지정합니다."^].

.성능 향상
데이터 분류의 모든 페이지, 데이터 매핑 보고서 및 조사 페이지의 필터 로딩 시간이 단축되었습니다.

.조사 보고서 작성 예상 시간
조사 보고서를 다운로드할 때, 이제 데이터 분류 기능에 다운로드 완료 예상 시간이 표시됩니다.
