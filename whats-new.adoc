== 2026년 3월 10일



=== Cloud Volumes ONTAP



==== Cloud Volumes ONTAP에 대한 Console 에이전트 프록시 설정을 관리하는 기능

이제 연결이 끊어지거나 프록시 설정이 잘못된 경우에도 NetApp Console 에이전트를 사용하여 Cloud Volumes ONTAP의 프록시 설정을 관리할 수 있습니다. 이전에는 Console 에이전트가 20분 동안 Cloud Volumes ONTAP에 연결할 수 없는 경우 수동 프록시 설정을 기본 설정으로 덮어썼습니다. 이로 인해 AutoSupport 메시지 관련 문제를 포함한 통신 오류가 발생했습니다. 기존 시스템의 프록시 설정을 유지하려면 다음 API 호출을 실행하십시오.

`PUT /occm/config`

요청 본문에 다음 매개변수를 포함하십시오.

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
기본 모드는 표준 모드입니다. 즉, Console 에이전트가 20분 동안 Cloud Volumes ONTAP에 연결할 수 없는 경우 프록시 설정을 기본값으로 덮어씁니다.

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["수정 가능한 NetApp Console 매개변수를 구성합니다"^]

=== 백업 및 복구



==== ONTAP Volumes 워크로드 개선 사항

이번 릴리스에서 ONTAP Volumes 워크로드는 다음과 같은 향상된 기능을 제공합니다.

* *원본 위치로 복원*: 이제 로컬 스냅샷 및 복제된 스냅샷의 파일을 원본 위치뿐만 아니라 다른 위치로도 복원할 수 있습니다.
* *시간별 볼륨 검색*: 이제 Dashboard에서 새로운 볼륨을 더 빠르게 검색하고 새로 고칩니다.
* *다크 사이트 지원*: 이제 다크 사이트용 Backup and Recovery를 https://mysupport.netapp.com/site/products/all/details/netapp-console/downloads-tab/download/61906/4.4.0/downloads["NetApp 지원 사이트"]에서 다운로드할 수 있습니다. 현재 다크 사이트용 NetApp Backup and Recovery는 ONTAP Volumes 워크로드만 지원합니다.
* *Unicode 호환성 개선*: 카탈로그 색인에서 Unicode 문자를 더 잘 처리합니다.
* *FlexGroup 호환성 향상*: ONTAP FlexVol 볼륨을 FlexGroup 볼륨으로 변환할 때 이제 변환 전에 생성된 백업을 복원할 수 있습니다.
* *다중 버킷 지원*: 이제 시스템 내 볼륨을 서로 다른 클라우드 공급자에 걸쳐 시스템당 최대 6개의 버킷으로 보호할 수 있습니다.


ONTAP Volume 워크로드 보호에 대한 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/prev-ontap-protect-overview.html["ONTAP 볼륨 워크로드 보호 개요"]를 참조하십시오.



==== VMware 워크로드 개선 사항

이번 릴리스에서 VMware 워크로드는 다음과 같은 향상된 기능을 소개합니다.

* *객체 저장소에서 게스트 데이터 복원*: 이제 객체 저장소에서 게스트 파일과 폴더를 복원할 수 있습니다.
* *VMFS/NFS 빠른 복원*: 이제 다른 위치로 복원할 때 빠른 복원 옵션을 활성화할 수 있습니다. 이 옵션을 활성화하면 오브젝트 저장소에서 VM 메타데이터(파일, LUN 및 네임스페이스)만 검색되므로 전체 복원보다 볼륨을 더 빠르게 사용할 수 있습니다.
* *Amazon FSx for NetApp ONTAP 지원*: VMware 워크로드에서 이제 버전 9.13.1 이상을 실행하는 Amazon FSx for NetApp ONTAP 스토리지 시스템을 지원합니다.


VMware 워크로드 보호에 대한 자세한 내용은 https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-vmware-protect-overview.html["VMware 워크로드 보호 개요"]을 참조하십시오.



=== 데이터 분류



==== 버전 1.52

.리소스 유효성 검사
안정성을 보장하기 위해 Data Classification은 스캔, 보고서 생성 및 작업 전반에 걸쳐 사용 가능한 메모리 및 디스크 리소스를 검증합니다. 메모리 사용량이 95%를 초과하거나 디스크 공간이 6GB 미만으로 떨어지면 Data Classification은 자동으로 작업을 일시 중지합니다. 리소스가 충분해지면 작업이 재개됩니다.

시스템 리소스를 모니터링하려면 link:https://docs.netapp.com/us-en/data-services-data-classification/task-health-monitor.html["Data Classification의 상태를 모니터링하세요"^]을 참조하십시오.

.복사 및 이동 제한 증가
이제 CIFS 대상에 대한 복사 및 이동 기능이 지원됩니다. 두 작업 모두 파일당 크기 제한이 기존 50MB에서 1GB로 증가했습니다. 두 작업 모두 작업량 제한이 1TB 또는 500만 파일로 늘어났습니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-data-classification/task-investigate-data.html["데이터 조사"^]을 참조하십시오.

.Active Directory 통합 구성 권한
Data Classification에서 Active Directory(AD) 통합 설정 변경은 이제 관리자에게만 허용됩니다. 뷰어는 더 이상 AD 설정을 수정할 수 없습니다.

분류 뷰어 역할에 대한 자세한 내용은 link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html#data-service-roles["데이터 서비스 역할"^]을 참조하십시오.

.투명 프록시 지원
Data Classification은 이제 투명 프록시 구성을 지원하여 프록시 또는 제한된 네트워크 환경에서 더욱 원활한 배포를 가능하게 합니다.

제한된 환경에서의 배포에 대한 자세한 내용은 link:https://docs.netapp.com/us-en/data-services-data-classification/task-deploy-compliance-dark-site.html["인터넷 연결이 없는 Linux 호스트에 NetApp Data Classification 설치"^]을 참조하십시오.

재해 복구 기능에서 vCenter 암호에 대한 암호 제한을 해제하여 공백이 포함된 암호를 허용하도록 했습니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-disaster-recovery/use/drplan-create.html#modify-a-replication-plan["복제 계획 수정"^]를 참조하십시오.

=== 랜섬웨어 복원력



==== 사용자 행동 알림에서 여러 사용자 일괄 제외

Ransomware Resilience는 이제 기존의 UI 및 CSV 기반 옵션 외에도 사용자 행동 활동 알림에서 제외할 사용자 목록을 JSON 형식으로 업로드하는 기능을 지원합니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/suspicious-user-activity.html#exclude-users-from-alerts["알림에서 사용자 제외"^]를 참조하십시오.



==== 사용자 행동 활동 개선

Ransomware Resilience는 이메일 알림을 추가하고 지원되는 SIEM 이벤트 데이터(AWS Security Hub, Microsoft Sentinel 및 Splunk Cloud)에 사용자 행동 이벤트를 포함하여 사용자 행동에 대한 인사이트를 개선했습니다. 이러한 개선 사항은 보안 및 규정 준수 모니터링에 대한 보다 정확한 인사이트를 제공합니다.

자세한 내용은 link:https://docs.netapp.com/us-en/data-services-ransomware-resilience/suspicious-user-activity.html["NetApp Ransomware Resilience를 SIEM에 연결하세요"^]를 참조하십시오.

=== Keystone

NetApp Console 의 Keystone 대시보드의 새로운 기능 향상:



==== 클러스터의 볼륨 탭의 새 열

*자산*의 *클러스터별 볼륨* 탭에 *핫 데이터*, *콜드 티어(%)*, *핫 티어(%)*라는 세 개의 새로운 열이 추가되었습니다. 더 자세한 내용을 알아보려면 link:./integrations/assets-tab.html["Keystone 구독과 관련된 자산 보기"] 및 link:./integrations/assets.html["Keystone 구독 전반에 걸쳐 자산을 확인하세요."]을 참조하십시오.



==== 소비 추세에서 청구된, 청구되지 않은 및 임시 버스트 사용량

*소비 추세* 탭의 누적 버스트 청구 그래프에 이제 청구된 버스트 사용량, 청구되지 않은 버스트 사용량 및 잠정 버스트 사용량이 표시됩니다. 자세한 내용은 link:./integrations/consumption-tab.html#view-accrued-burst["누적된 버스트 보기"]를 참조하십시오.



==== 모니터링 보기 개선 사항

*Monitoring* 보기에는 다음과 같은 업데이트가 포함되어 있습니다.

* *Alerts 탭*: *Alert* 열의 이름이 *Alert type & condition*으로 변경되었습니다. 이제 *Status* 열 옆에 있는 점 세 개 아이콘을 사용하여 알림을 편집하거나 메모를 추가하거나 알림 세부 정보를 볼 수 있습니다.
* *Alert monitors 탭*: *Condition* 열의 이름이 *Monitor type & condition*으로 변경되었습니다.


더 자세한 내용을 알아보려면 link:./integrations/monitoring-alerts.html["Keystone 구독 관련 알림을 확인하고 관리하세요."] 및 link:./integrations/monitoring-alert-monitors.html["Keystone 구독에 대한 알림 모니터를 보고 생성하세요."]을 참조하십시오.

=== Cloud Volumes ONTAP



==== 차세대 Google Cloud VM을 위한 Cloud Volumes ONTAP 지원

9.18.1에서 NetApp은 새로운 Cloud Volumes ONTAP 배포를 N2 VM에서 차세대 Google Cloud C3 시리즈 VM으로 전환하여 더욱 빠르고 확장 가능한 환경을 제공합니다. 이제 Google Cloud에서 Cloud Volumes ONTAP 9.18.1 이상 버전을 배포할 때 C3 시리즈 VM의 이점을 활용할 수 있습니다. C3 시리즈 머신은 Google Virtual NIC(gVNIC)와 Hyperdisk Balanced 디스크를 사용하여 향상된 성능과 더 높은 용량 제한을 제공하며, 집약적인 워크로드에 필요한 동적 성능을 보장합니다.


NOTE: 현재 Cloud Volumes ONTAP는 단일 노드 배포 환경에서만 C3 시리즈를 지원합니다.

Cloud Volumes ONTAP 시스템이 9.18.1 이상을 실행하는 경우 간편한 단일 노드 배포에 사용하는 사전 구성된 패키지는 자동으로 C3 VM을 사용하며 워크로드 요구 사항에 따라 IOPS 및 처리량 매개 변수를 사용자 지정할 수 있습니다. 마찬가지로 애그리게이트를 생성하는 동안 Hyperdisk Balanced 디스크를 추가하여 Google Cloud에서 더 나은 성능과 확장성을 달성할 수 있습니다. 또한 기본 Flash Cache 지원을 위해 C3 시리즈 머신의 LSSD 변형을 선택할 수 있습니다.

C3 VM은 Hyperdisk Balanced 디스크만 지원하기 때문에 애그리게이트에 볼륨을 추가할 때 디스크 유형을 변경할 수 없습니다. 마찬가지로 N2 VM 유형의 시스템을 C3 VM으로 복제할 때 디스크 유형은 기본적으로 Hyperdisk Balanced로 설정됩니다.

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud에서 Cloud Volumes ONTAP에 대해 지원되는 구성"^]

https://docs.cloud.google.com/compute/docs/general-purpose-machines#c3_series["Google 문서: C3 머신 시리즈"^]



==== Azure의 Cloud Volumes ONTAP용 VNet 보안

Azure 단일 및 다중 가용성 영역에 배포된 Cloud Volumes ONTAP 9.18.1 이상 버전은 전송 중인 데이터를 보호하기 위한 계층형 보안 전략의 일환으로 Azure 가상 네트워크(VNet) 암호화를 지원합니다. Cloud Volumes ONTAP는 Azure의 기본 데이터그램 전송 계층 보안(DTLS) 프로토콜을 활용하여 ONTAP 노드, 관리 인터페이스 및 기타 Azure 서비스 간의 통신을 보호하고, 데이터 가로채기 및 무단 액세스를 방지합니다. 이러한 네트워크 수준 암호화는 ONTAP에 내장된 스토리지 및 저장 데이터 보호 기능을 보완하여 데이터에 대한 엔드 투 엔드 보안을 제공합니다.

https://review.docs.netapp.com/us-en/storage-management-cloud-volumes-ontap_gcp-sn-c3-support/reference-networking-azure.html#networking-for-azure-vnet-encryption["Azure VNet 암호화를 위한 네트워킹"^]

=== Cloud Volumes ONTAP



==== Azure에서 Ebdsv5 및 E104ids_v5 VM 지원

Digital Advisor에는 이제 스토리지 환경 전반의 ONTAP 보안 위험에 대한 통합 보기를 제공하는 보안 보고서가 포함되어 있습니다. 이 보고서를 통해 클러스터, SVM 및 볼륨 전반에 걸쳐 잘못된 구성, 규정 준수 격차 및 보안 문제를 더 빠르게 감지하여 안전한 스토리지 인프라를 유지할 수 있습니다.

Upgrade Advisor는 이제 동일한 ONTAP 주요 릴리스 내에서 노드가 서로 다른 패치 레벨을 실행하는 클러스터에 대한 업그레이드 보고서를 생성합니다. 이 개선 사항을 통해 고객은 클러스터가 주요 버전 간에 혼합되지 않는 한 원활하게 업그레이드를 진행할 수 있으며, 업그레이드 계획에 있어 더 큰 유연성을 제공합니다.



==== 지원 기간이 종료된 ONTAP 버전에 대한 지원이 확장되었습니다.

Upgrade Advisor는 이제 제한적 지원이 종료된 ONTAP 버전에 대한 업그레이드 계획 생성을 지원하며, 9개월의 유예 기간을 제공합니다. 이 기능은 전환 기간 동안 명확한 지침을 제공하여 고객이 적시에 조치를 취하고 업그레이드 중단을 방지할 수 있도록 지원합니다.

==== NetApp Console 관리

이 릴리스에는 다음이 포함됩니다.

.NetApp Console 에서 페더레이션 그룹에 대한 역할 기반 액세스
NetApp Console 페더레이션 그룹(예: Active Directory 그룹)에 역할을 할당하는 기능을 지원하므로 관리자는 조직의 ID 공급자(IdP)의 그룹 멤버십을 기반으로 사용자 온보딩 및 오프보딩을 자동화할 수 있습니다. 이 기능은 관리 부담을 줄이고 그룹 멤버십이 변경될 때 콘솔 액세스 권한을 즉시 업데이트하여 안전하고 일관된 액세스를 보장합니다.

link:https://docs.netapp.com/us-en/console-setup-admin/task-iam-manage-members-permissions.html#add-a-federated-group-to-your-organization["조직 구성원에게 연합 그룹에 대한 액세스 권한을 부여하는 방법을 알아보세요."]

.NetApp Console 제한 모드일 때 페더레이션 지원
이제 제한 모드로 설정된 NetApp Console 조직에 대해 페더레이션을 활성화할 수 있습니다. 이를 통해 사용자는 회사 자격 증명을 사용하여 콘솔에 로그인하는 동시에 제한 모드의 보안 이점을 유지할 수 있습니다.

.읽기 전용 모드
NetApp Console 조직을 읽기 전용 모드로 설정할 수 있습니다. 읽기 전용 모드에서는 사용자가 리소스와 설정을 볼 수는 있지만 변경할 수는 없습니다. 조직 관리자 또는 최고 관리자는 조직에 대해 읽기 전용 모드를 활성화할 수 있습니다. 읽기 전용 모드가 활성화된 경우, 관리자 권한을 가진 사용자는 필요에 따라 변경 작업을 수행하기 위해 수동으로 권한을 상승시켜야 합니다.

link:https://docs.netapp.com/us-en/console-setup-admin/task-enable-read-only-mode.html["콘솔 조직에 대해 읽기 전용 모드를 활성화하는 방법을 알아보세요."]

link:https://docs.netapp.com/us-en/console-setup-admin/task-user-settings.html["조직이 읽기 전용 모드일 때 자신의 역할을 강화하는 방법을 알아보세요."]

=== Cloud Volumes ONTAP



==== Cloud Volumes ONTAP 에 선호하는 결제 옵션

이제 Cloud Volumes ONTAP 사용량 및 초과 사용량 계산에 사용할 원하는 청구 옵션을 선택할 수 있습니다. 2025년 6월 25일부터 BYOL(Bring Your Own Licenses) 라이선스 모델의 사용이 제한됨에 따라 NetApp NetApp Console 의 *라이선스 및 구독* 섹션에 선호하는 요금 청구 방식을 추가했습니다. 청구 및 초과 사용량 처리에는 연간 마켓플레이스 구독을 사용하거나 기존의 BYOL 모델을 사용하는 것이 좋습니다. BYOL 모델을 선호하는 옵션으로 선택할 수 있습니다. 이를 통해 조직의 재정 전략 및 사용 패턴에 가장 적합한 요금 청구 방식을 유연하게 선택할 수 있습니다.

https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/concept-licensing-charging.html#billing-preferences-and-overages["요금 설정 및 초과 사용 요금"^].
